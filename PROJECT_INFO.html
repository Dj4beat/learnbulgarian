What This Project Is Doing

The later lessons are being upgraded one day at a time so they match the stronger early lessons in teaching quality, Bulgarian naturalness, and audio usefulness. The model lesson for this phase is day16.html.

The work should not be treated as finished when a lesson merely has a few grouped audio buttons. The goal is a fully improved lesson: better Bulgarian, better teaching, more examples, richer topic coverage, new teaching elements where needed, and dense useful audio.

How The Audio Is Created

The course pages use data-text attributes on audio buttons. The generator script scans those values and creates MP3 files for each target phrase.

Generator script

generate_pro_audio.py

What it expects

  • HTML pages with play-audio buttons and data-text values
  • A working Python environment in venv
  • Running from the project folder on Windows

Where audio goes

Generated files are written into audio/<page-name>/.

Important filename rule

The page JavaScript and the Python generator must sanitize Bulgarian text the same way. If they do not, the site will miss the MP3 file and silently fall back to browser speech synthesis.

Command

cd /d "C:\Users\Dj4be\Desktop\New Bulgarian"
venv\Scripts\python.exe generate_pro_audio.py day17.html

Multi-file example

venv\Scripts\python.exe generate_pro_audio.py day43.html day49.html day52.html

Force regeneration

If an MP3 already exists and the lesson text has been improved, use --force to overwrite the old file. This is important for fixing stale audio.

venv\Scripts\python.exe generate_pro_audio.py --force day01.html

Quality Rules

  • Review every lesson as if checking work for a native Bulgarian teacher teaching English-speaking adults.
  • Reject Russian intrusions, false friends, awkward calques, and unnatural Bulgarian.
  • Keep CEFR wording correct: A2 = Pre-intermediate, B1 = Intermediate.
  • Use day16.html as the standard for audio density and pedagogical richness.
  • Add new teaching elements if the page needs them: extra examples, mini-dialogues, contrast notes, grouped practice, or stronger recap material.
  • Prefer generous topic examples over minimal coverage. Visitors value lots of useful examples tied to the lesson topic.
  • Lessons must also pass an engagement check: they should not feel like flat word lists.
  • Add Bulgarian-specific facts, customs, etiquette, or practical local details where they genuinely help the learner.
  • After every meaningful change, update both PROJECT_STATUS.md and this file.

Engagement Standard

The course now has an explicit engagement requirement in addition to accuracy and audio coverage.

  • Use ENGAGEMENT_AGENT.md to review whether a lesson feels interesting, useful, and connected to Bulgarian life.
  • Prefer memorable examples, reusable speaking chunks, and realistic scenarios over raw vocabulary-only sections.
  • Add Bulgarian context when it helps: customs, expectations, travel detail, etiquette, or local cultural habits.
  • Previously rebuilt lessons also need an engagement pass if they still feel too list-like.
  • Every module should aim to be informative, entertaining, and educational, not just correct and comprehensive.
  • Where the topic allows it, lessons should begin with a proper scene-setting introduction before Section 1 rather than jumping straight into lists.
