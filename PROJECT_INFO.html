🎙️ Audio Generation

Two scripts exist — only one works.
✅ generate_pro_audio.py — use this always
❌ generate_audio.py — do NOT run this. It is an abandoned prototype using gemini-2.5-flash-tts which does not exist as a valid model. It fails on every call. The hardcoded API key it contained has been revoked.

How generate_pro_audio.py works

  • Every play button in the HTML has a data-text attribute containing Bulgarian text
  • generate_pro_audio.py scans those values and calls Google Cloud TTS
  • Voice: bg-BG-Standard-A (female Bulgarian)
  • MP3 files are saved to audio/<page-name>/
  • The page JavaScript tries the MP3 first; falls back to browser Web Speech API only if the file is missing
  • Uses a Google Cloud service account credential file — not the GOOGLE_API_KEY environment variable
Always run from the project folder using the venv Python — not the system Python.

Basic command — generate new audio

cd /d "C:\Users\Dj4be\Desktop\New Bulgarian"
venv\Scripts\python.exe generate_pro_audio.py --force day17.html

Multiple files at once

venv\Scripts\python.exe generate_pro_audio.py --force day17.html day18.html day19.html

When to use --force

Always use --force if you have edited any data-text values in the HTML. Without it, existing MP3 files are skipped even if the text changed.

Critical: Cyrillic filename rule

The browser's makeFilename() function and the Python generator must sanitise text identically. If they differ, pages silently fall back to speech synthesis instead of playing the stored MP3. This was fixed in the shared JS across all pages — do not alter the filename logic without updating both sides.

Full workflow for amending a day

  1. Edit the root dayXX.html
  2. Regenerate audio: venv\Scripts\python.exe generate_pro_audio.py --force dayXX.html
  3. Upload HTML to server: /app811033055/bulgarian/dayXX.html
  4. Upload audio folder to server: /app811033055/bulgarian/audio/dayXX/
  5. Commit to git: git add dayXX.html audio/dayXX/ && git commit -m "Description"
  6. Update PROJECT_STATUS.md

📊 CEFR Level Labels

The CEFR (Common European Framework of Reference) was created after decades of level-naming practice in English language teaching. There is no perfect one-to-one correspondence between CEFR codes and the traditional ELT level names. Use these as the authoritative mapping:

CEFR LevelCorrect LabelIn badges / nav
A1Beginner / ElementaryA1 — Beginner
A2Elementary / Pre-intermediateA2 — Pre-intermediate
B1Pre-intermediate / IntermediateB1 — Pre-intermediate / Intermediate
A2 is NOT Intermediate. This was a common error in early versions. A2 straddles Elementary and Pre-intermediate. B1 straddles Pre-intermediate and Intermediate. Both labels are now correct across all 60 day files.

📐 Quality Rules

Language accuracy

  • Review as a native Bulgarian teacher
  • Reject Russian words, false friends, awkward calques
  • Unnatural Bulgarian must be corrected, not left in

CEFR labels

  • A2 = Pre-intermediate (never "Intermediate")
  • B1 = Pre-intermediate / Intermediate (never "Advanced")
  • Check badges, optgroups, meta descriptions

Audio

  • Every table row should have a data-text button
  • Use --force when regenerating edited dialogue
  • Upload audio/ folder after regeneration

Engagement

  • Not a flat word list — add scene context
  • Include Bulgarian customs, local detail, practical tips
  • Every lesson should feel useful and interesting

Mobile

  • Treat as mobile-first, not desktop-first
  • Tables stack as cards on phones
  • Audio buttons must stay visible and tappable

Currency (from Jan 2026)

  • All prices in euros — лева is history
  • 1 EUR = 1.95583 BGN (irrevocable rate)
  • Euro subunit: евро/цент or евро/стотинки

✏️ How to Make Amendments

Editing a lesson (text only — no audio change)

  1. Edit the root dayXX.html file
  2. Upload to server:
    curl --insecure -T dayXX.html \
  "sftp://access811018624.webspace-data.io/app811033055/bulgarian/dayXX.html" \
  --user "u99549562:United!!001ad"
  3. Commit: git add dayXX.html && git commit -m "Description"

Editing a lesson (changing data-text values)

  1. Edit the root dayXX.html
  2. Regenerate audio: 
    cd /d "C:\Users\Dj4be\Desktop\New Bulgarian"
venv\Scripts\python.exe generate_pro_audio.py --force dayXX.html
  3. Upload HTML and audio folder using paramiko script
  4. Commit: git add dayXX.html audio/dayXX/ && git commit -m "Description"

Adding a new audio button

Add a button with class play-audio and data-text (Bulgarian) and data-phonetic (romanisation) attributes:

<button class="play-audio audio-controls"
  data-text="Добър ден"
  data-phonetic="Dobar den">▶ Play</button>

Then run generate_pro_audio.py --force dayXX.html to create the MP3.

Adding a phrase row

<div class="phrase-row">
  <div class="phrase-main">
    <span class="phrase-bg" lang="bg">Добър ден.</span>
    <span class="phrase-rom">Dobar den.</span>
    <span class="phrase-en">Good afternoon.</span>
  </div>
  <button class="play-inline play-audio audio-controls"
    data-text="Добър ден."
    data-phonetic="Dobar den.">▶ Listen</button>
</div>

python -c "
import re, sys
